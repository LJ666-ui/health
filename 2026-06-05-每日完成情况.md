### 任务6：Maven构建问题排查与解决（🔥 核心突破）⭐⭐⭐

#### 问题现象
```bash
# 使用系统自带Maven 3.0.5编译失败
[ERROR] Failed to execute goal org.apache.maven.plugins:maven-compiler-plugin:3.11.0:compile
[ERROR] The plugin requires Maven version 3.2.5 -> [Help 1]
```

#### 尝试的解决方案（均因网络问题失败）
| 方案 | 命令 | 结果 |
|------|------|------|
| 下载Maven 3.9.6（Apache官方源） | curl/wget dlcdn.apache.org | ❌ gzip格式错误 |
| 下载Maven 3.9.6（清华镜像） | wget mirrors.tuna.tsinghua.edu.cn | ❌ 403 Forbidden |
| 下载Maven 3.9.6（阿里云镜像） | wget mirrors.aliyun.com | ❌ 403 Forbidden |
| yum安装maven | yum install -y maven | ❌ 版本仍为3.0.5太旧 |

#### 最终解决方案：使用Docker容器运行Maven ✅
```bash
docker run --rm \
  -v "$(pwd)/zhihealth-cloud":/usr/src/maven \
  -w /usr/src/maven \
  maven:3.9-eclipse-temurin-17 \
  mvn clean package -DskipTests
```

**优势：**
- 不需要本地安装Maven
- 自动使用最新版Maven 3.9 + JDK 17
- 构建完自动清理容器（--rm）
- Docker Hub拉取镜像成功（网络通畅）

## 🔧 遇到的问题及解决方案

### 问题1：VNC无法上传文件
**现象：** VNC执行`rz`后只显示`CCCCC...`等待，不弹窗
**原因：** 阿里云VNC控制台不支持文件传输功能
**解决：** 改用Git clone方案，绕过文件上传

### 问题2：PowerShell scp连接失败
**现象：** 从本地PowerShell执行scp报连接错误
**原因：** SSH端口未开放或网络限制
**解决：** 放弃scp，统一使用Git方案

### 问题3：Maven下载全部失败（4次尝试）
**现象：** Apache官方源/清华镜像/阿里云镜像全部返回gzip错误或403
**原因：** 服务器网络对HTTP大文件下载有限制
**解决：** 使用Docker容器内嵌Maven（`maven:3.9-eclipse-temurin-17`），通过Docker Hub拉取镜像成功

### 问题4：docker compose找不到配置文件
**现象：** `docker compose up -d` 报 "no configuration file provided"
**原因：** 当前目录下没有默认的docker-compose.yml，需要指定-f参数
**解决：** `docker compose -f docker-compose.master.yml up -d`

## 💡 经验总结

1. **Docker作为构建环境的优势**：当服务器软件版本过旧或网络受限时，用Docker容器运行构建工具是最佳替代方案
2. **Git优于手动传文件**：对于频繁更新的项目，Git方式的增量同步远比每次全量压缩上传高效
3. **VNC局限性**：阿里云VNC仅适合紧急运维，日常开发应使用SSH客户端（如XShell/MobaXterm/PuTTY）
4. **restart: always的重要性**：Docker容器的重启策略确保服务器重启后服务自动恢复，无需人工干预

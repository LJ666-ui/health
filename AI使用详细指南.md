# 智康云枢 - AI使用详细指南（修订版）

> 核心原则：**代码类AI基本都能帮，操作性任务AI只能写命令/脚本但执行必须自己来，论文内容AI只能润色不能代写**

---

## 一、AI可用性三级分类（先看这个！）

| 标记 | 含义 | 你要做什么 |
|------|------|-----------|
| ✅可用AI | AI可以直接帮你完成 | 你描述需求→AI生成代码/配置→你理解后复制使用 |
| ⚠️AI辅助有限 | AI只能帮你一部分 | AI帮你写代码/命令，但**执行、观察、判断必须你自己来** |
| ❌不可用AI | 必须你自己做 | AI完全无法替代，必须自己动手操作 |

---

## 二、对谁使用AI？（工具选择）

| 你要做什么 | 用什么工具 | 为什么 |
|-----------|-----------|--------|
| 写Java/Python代码 | **Trae IDE**（就是你现在用的） | 直接在编辑器里生成代码，最方便 |
| 写SQL、设计表结构 | **Trae IDE** 或 **ChatGPT网页版** | 都可以，Trae更方便直接写入文件 |
| 写配置文件（YAML/Docker/Nginx） | **ChatGPT网页版** | 配置文件生成准确率高，方便复制粘贴 |
| 讨论架构方案、算法选型 | **ChatGPT网页版** | 适合来回讨论，不涉及直接写代码 |
| Bug排查 | **Trae IDE** | 可以直接看代码上下文，分析更准 |
| 论文语言润色 | **ChatGPT网页版** 或 **Claude** | 中文表达优化效果好 |
| 英文摘要翻译 | **ChatGPT网页版** | 学术翻译比较准确 |

**对谁用**：Trae IDE（直接在编辑器里让AI生成代码）

**提示词模板**：

```

- 项目使用Spring Boot 2.7.x + Spring Cloud 2021.x + MyBatis-Plus
- 数据库是MySQL 8.0

- 实体类字段：[列出字段，如：id, username, password, role, createTime]
- 要求：使用RESTful风格，统一响应格式Result<T>，包含参数校验
```

**具体示例**：

```
请帮我编写用户注册接口：
- 项目使用Spring Boot 2.7.x + MyBatis-Plus
- 实体类User字段：id(Long), username(String), password(String), phone(String), role(Integer), createTime(LocalDateTime)
```

### 3.2 Python数据处理/机器学习代码编写

**对谁用**：Trae IDE

**提示词模板**：

```
请帮我用Python编写[功能名称]代码：
- 使用库：[Pandas/NumPy/scikit-learn/PyTorch等]
- 数据来源：[InfluxDB/MySQL/MongoDB/CSV文件]
- 数据格式：[描述数据结构，如：DataFrame包含heart_rate, blood_pressure, temperature列]
- 处理逻辑：[详细描述，如：去除心率>200的异常值，缺失值用前值填充，归一化到0-1]
- 输出：[处理后的数据格式]
```

**具体示例 - 数据清洗**：

```
请帮我用Python编写健康数据清洗代码：
- 使用Pandas
- 数据来源：从InfluxDB查询的健康指标时序数据
- 数据格式：DataFrame包含time, heart_rate, blood_pressure_sys, blood_pressure_dia, temperature列
- 清洗逻辑：
  1. 去除心率<30或>220的异常值
  2. 去除收缩压<60或>260的异常值
  3. 缺失值用前向填充
  4. 对数值列做Min-Max归一化
- 输出：清洗后的DataFrame
```

**具体示例 - 随机森林模型**：

```
请帮我用scikit-learn编写随机森林回归模型，预测健康数据趋势：
- 输入特征：过去7天的心率、血压、体温统计值（均值、标准差、最大值、最小值）
- 输出：未来1天的心率预测值
- 要求：划分训练集/测试集(8:2)，使用RandomForestRegressor，n_estimators=100
- 评估指标：MAE、RMSE、R²
```

**具体示例 - LSTM模型**：

```
请帮我用PyTorch搭建LSTM时序预测模型：
- 输入：过去24小时的健康指标时序数据（心率、血压、体温），shape=(batch, seq_len=24, features=3)
- 网络结构：LSTM(2层, hidden_size=128) → 全连接层 → 输出(预测未来1小时的3个指标)
- 训练配置：MSELoss + Adam(lr=0.001) + 100 epochs
- 要求：包含训练循环、验证、保存最优模型权重
```

## 四、⚠️AI辅助有限的任务（重点看！操作性任务怎么用AI）

### 4.1 服务器操作（Docker部署、服务启动等）

**核心原则**：AI帮你**写命令和配置**，你负责**执行和观察**

**具体流程**：

| 步骤 | 谁来做 | 做什么 |
|------|--------|--------|
| 1. 写Dockerfile | AI | 你给AI描述需求，AI生成Dockerfile |
| 2. 写docker-compose.yml | AI | 你列出服务清单，AI生成配置 |
| 3. 上传文件到服务器 | **你自己** | 用Xftp/SCP上传 |
| 4. SSH登录服务器 | **你自己** | `ssh root@服务器IP` |
| 5. 执行docker build | **你自己** | 复制AI给的命令，在服务器执行 |
| 6. 执行docker-compose up | **你自己** | 在服务器执行 |
| 7. 查看容器状态 | **你自己** | `docker ps` 查看是否正常运行 |
| 8. 查看日志排查问题 | **你自己** | `docker logs 容器名` 查看报错 |
| 9. 把报错给AI分析 | AI | 你把日志贴给AI，AI帮你分析原因 |
| 10. 修改后重新部署 | AI+你 | AI给修改方案，你执行 |

**实操示例 - 部署网关服务**：

```
第1步（问AI）：
"请帮我编写zhihealth-gateway的Dockerfile，基于openjdk:8-jre-alpine，Jar包是target/zhihealth-gateway.jar，端口8081，JVM参数-Xmx256m"

第2步（你自己）：
- 把AI生成的Dockerfile保存到项目目录
- 用Xftp上传Jar包和Dockerfile到腾讯云服务器 /opt/zhihealth/gateway/ 目录

第3步（你自己）：
- SSH登录腾讯云：ssh root@43.xxx.xxx.xxx
- 进入目录：cd /opt/zhihealth/gateway/
- 构建镜像：docker build -t zhihealth-gateway:latest .
- 启动容器：docker run -d --name gateway -p 8081:8081 zhihealth-gateway:latest

第4步（你自己）：
- 检查状态：docker ps（看gateway是否在运行）
- 查看日志：docker logs gateway（看有没有报错）

第5步（如果有报错，问AI）：
"我的gateway容器启动失败，日志如下：[粘贴docker logs输出]，请帮我分析原因"
```

### 4.2 模型训练

**核心原则**：AI帮你**写模型代码和训练脚本**，你负责**运行训练和判断效果**

| 步骤 | 谁来做 | 做什么 |
|------|--------|--------|
| 1. 编写模型定义代码 | AI | 你描述网络结构，AI生成PyTorch代码 |
| 2. 编写训练脚本 | AI | 你描述训练配置，AI生成训练循环 |
| 3. 运行训练 | **你自己** | `python train.py` |
| 4. 观察Loss曲线 | **你自己** | 看Loss是否下降，是否收敛 |
| 5. 如果Loss不下降 | AI+你 | 把训练日志贴给AI，AI分析原因 |
| 6. 调整参数重新训练 | AI+你 | AI建议参数，你修改后重新运行 |
| 7. 评估模型效果 | **你自己** | 运行评估脚本，看准确率/RMSE等指标 |

**实操示例 - 训练LSTM模型**：

```
第1步（问AI）：
"请帮我用PyTorch编写LSTM时序预测模型代码，2层LSTM 128隐藏单元，输入3个特征，输出3个预测值，包含训练循环和验证逻辑"

第2步（你自己）：
- 把AI生成的代码保存为train_lstm.py
- 准备训练数据
- 运行：python train_lstm.py

第3步（你自己观察）：
- 看输出：Epoch 1 Loss: 0.5234, Epoch 2 Loss: 0.4123, ...
- 判断：Loss是否在下降？下降是否足够快？
- 如果Loss一直不下降，说明有问题

第4步（如果有问题，问AI）：
"我的LSTM训练10个epoch后Loss从0.5降到0.48就不再下降了，数据是健康指标时序数据，可能是什么原因？如何调整？"

第5步（根据AI建议调整）：
- AI可能建议：降低学习率、增加epoch数、调整batch_size
- 你修改代码后重新运行
```

## 五、❌不可用AI的任务（必须自己做）

| 任务 | 为什么不能用AI | 你应该怎么做 |

## 七、最实用的AI使用工作流

### 写代码的标准流程

```
1. 你：向AI描述需求（用上面的提示词模板）
2. AI：生成代码
3. 你：阅读理解代码（确保能解释每一行）
4. 你：复制代码到项目中
5. 你：运行测试
6. 如果报错：把报错信息贴给AI，让AI帮你分析
7. AI：给出修改建议
8. 你：修改后重新测试
```

### 部署操作的标准流程

```
1. 你：向AI描述部署需求
2. AI：生成Dockerfile/docker-compose.yml/Nginx配置
3. 你：保存配置文件
4. 你：用Xftp上传到服务器
5. 你：SSH登录服务器
6. 你：执行docker build / docker-compose up
7. 你：docker ps 检查状态
8. 如果报错：docker logs 查看日志 → 贴给AI分析
9. AI：分析原因给修改方案
10. 你：修改后重新部署
```

### 模型训练的标准流程

```
1. 你：向AI描述模型结构和训练需求
2. AI：生成模型代码 + 训练脚本
3. 你：保存代码，准备数据
4. 你：python train.py 运行训练
5. 你：观察Loss是否下降
6. 如果Loss不下降：把训练日志贴给AI
7. AI：分析原因，建议调参方向
8. 你：调整参数重新训练
9. 你：在测试集上评估效果
```

## 五、安全架构设计

### 5.1 认证与授权

- JWT Token认证：用户登录后签发Token，前端请求携带Token
- Gateway统一鉴权：所有请求经过Gateway校验Token有效性
- RBAC权限模型：用户-角色-权限三级管控

### 5.2 数据安全

- 密码BCrypt加密存储
- 敏感数据脱敏展示
- 接口限流防刷（Redis + 令牌桶算法）
- SQL注入防护（MyBatis-Plus参数化查询）

### 5.3 传输安全

- HTTPS加密传输（生产环境）
- 跨域CORS统一配置
- 请求签名校验（关键接口）
                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         
## 六、技术难点与解决方案

| 难点 | 解决方案 |
|------|----------|
| 四库数据一致性 | Redis缓存+异步持久化+定时对账任务 |
| 大模型推理延迟 | 异步调用+结果缓存+流式响应 |
| 云服务器内存有限 | JVM参数调优(-Xmx256m)、轻量级镜像、按需启动 |
| 跨云服务调用延迟 | 本地缓存、减少跨云调用、批量传输 |
| 时序数据高效查询 | InfluxDB连续查询、降采样策略 |
| NLP中文处理精度 | Jieba分词+自定义健康词典+大模型语义理解 |
| 神经网络模型轻量化 | 模型量化(INT8)、知识蒸馏、小模型架构 |

## 🔥 今日核心突破：云服务器微服务构建

### 突破背景
- 服务器自带Maven版本过旧（3.0.5），无法编译Spring Boot 3.x项目（要求3.2.5+）
- 多次尝试下载Maven 3.9.x均因网络问题失败（Apache官方/清华/阿里镜像全部403或gzip错误）

### 解决方案：Docker容器Maven构建
```bash
docker run --rm \
  -v "$(pwd)/zhihealth-cloud":/usr/src/maven \
  -w /usr/src/maven \
  maven:3.9-eclipse-temurin-17 \
  mvn clean package -DskipTests
```

## 💡 关键经验记录

1. **Docker作为万能构建环境**：当目标环境的软件版本不满足需求时，Docker容器可以提供任意版本的构建工具链，且无需污染宿主机环境
2. **Git是云部署的最佳实践**：相比手动传文件，Git提供版本控制、增量同步、回滚能力、CI/CD集成等优势
3. **VNC的局限性**：阿里云ECS VNC仅适合紧急场景下的运维操作，日常开发应使用标准SSH客户端
4. **restart: always策略的价值**：确保服务器重启后所有关键服务自动恢复，是实现高可用性的基础配置
5. **国内网络对开源软件下载的影响**：Apache/清华/阿里镜像均可能存在访问限制，Docker Hub通常是更可靠的选择
